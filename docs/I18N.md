## 1. Qo'llab-quvvatlanadigan tillar

| Kod | Til | Holat |
|---|---|---|
| `uz` | O'zbek | Standart (default) |
| `ru` | Rus | Qo'llab-quvvatlanadi |

Boshqa til kodlari (masalan, `en`, `de`) `uz` ga fallback qiladi.

---

## 2. Til aniqlash tartibi

Har HTTP request uchun `LocaleMiddleware` quyidagi ustuvorlik bilan tilni aniqlaydi:

```
?lang= query parametri  →  Accept-Language header  →  uz (default)
```

**Misol:**

```
GET /catalog/products?lang=ru
Accept-Language: uz
```

Natija: `ru` (query parametr ustun turadi).

```
GET /catalog/products
Accept-Language: ru-RU,ru;q=0.9,en-US;q=0.8
```

Natija: `ru` (eng yuqori q-faktorli qo'llab-quvvatlanadigan til).

```
GET /catalog/products
Accept-Language: fr,de;q=0.9
```

Natija: `uz` (hech biri mos kelmadi — default).

Aniqlangan til `current_locale` `ContextVar`'ga yoziladi va request oxirida tozalanadi (async context izolyatsiyasi).

## 3. Xato envelope formati

Barcha xato javoblari quyidagi yagona formatda qaytadi:

```json
{
  "message_key": "auth.invalid_credentials",
  "message": "Telefon yoki parol noto'g'ri",
  "detail": null
}
```

| Maydon | Tip | Tavsif |
|---|---|---|
| `message_key` | `string` | Mashine-o'qiladigan kalit; UI/mobil tarjima logikasi uchun |
| `message` | `string` | Request tiliga mos lokalizatsiyalangan matn |
| `detail` | `any \| null` | Qo'shimcha ma'lumot (422 uchun Pydantic xato ro'yxati, boshqalarda `null`) |

**Bir xil xato uz va ru tilida:**

```bash
# O'zbek tili (default)
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone": "+998901234567", "password": "xato"}'
```

```json
{
  "message_key": "auth.invalid_credentials",
  "message": "Telefon yoki parol noto'g'ri",
  "detail": null
}
```

```bash
# Rus tili
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -H "Accept-Language: ru" \
  -d '{"phone": "+998901234567", "password": "xato"}'
```

```json
{
  "message_key": "auth.invalid_credentials",
  "message": "Неверный номер телефона или пароль",
  "detail": null
}
```

**422 validatsiya xatosi (detail ro'yxat):**

```bash
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -H "Accept-Language: ru" \
  -d '{}'
```

```json
{
  "message_key": "common.validation_error",
  "message": "Данные запроса некорректны",
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "phone"],
      "msg": "Field required"
    },
    {
      "type": "missing",
      "loc": ["body", "password"],
      "msg": "Field required"
    }
  ]
}
```

Xavfsizlik: `detail` ro'yxatidagi har bir elementdan `input`, `url`, `ctx` maydonlari olib tashlangan — parol/token qiymatlari 422 javobida echo qilinmaydi.

## 4. `message_key` katalogi

### auth.*

| `message_key` | uz | ru |
|---|---|---|
| `auth.invalid_credentials` | Telefon yoki parol noto'g'ri | Неверный номер телефона или пароль |
| `auth.inactive_user` | Hisob bloklangan | Аккаунт заблокирован |
| `auth.token_expired` | Token muddati tugagan | Срок действия токена истёк |
| `auth.token_invalid` | Token yaroqsiz | Токен недействителен |
| `auth.token_wrong_type` | Token turi noto'g'ri | Неверный тип токена |
| `auth.authentication_required` | Autentifikatsiya talab qilinadi | Требуется аутентификация |
| `auth.user_not_found` | Foydalanuvchi topilmadi | Пользователь не найден |

### rbac.*

| `message_key` | uz | ru |
|---|---|---|
| `rbac.permission_denied` | Ruxsat yo'q: `'{module}'` modulida `'{action}'` amali `'{role}'` roli uchun taqiqlangan | Доступ запрещён: действие `'{action}'` в модуле `'{module}'` недоступно для роли `'{role}'` |

`rbac.permission_denied` uchta format parametr qabul qiladi: `module`, `action`, `role`.

### common.*

| `message_key` | uz | ru |
|---|---|---|
| `common.not_found` | So'ralgan resurs topilmadi | Запрашиваемый ресурс не найден |
| `common.validation_error` | So'rov ma'lumotlari noto'g'ri | Данные запроса некорректны |
| `common.internal_error` | Ichki server xatosi | Внутренняя ошибка сервера |

## 5. `translate()` va `localized_name()` foydalanish namunalari

### `translate(key, locale, **params)`

```python
from app.core.messages import translate

# Aniq til
translate("auth.invalid_credentials", locale="ru")
# → "Неверный номер телефона или пароль"

# current_locale ContextVar (request ichida — middleware o'rnatadi)
translate("auth.invalid_credentials")
# → joriy request tiliga mos matn

# Format parametrlar bilan
translate(
    "rbac.permission_denied",
    locale="uz",
    module="catalog",
    action="delete",
    role="agent",
)
# → "Ruxsat yo'q: 'catalog' modulida 'delete' amali 'agent' roli uchun taqiqlangan"

# Noma'lum kalit — kalit o'zi qaytadi (WARNING log)
translate("nonexistent.key", locale="uz")
# → "nonexistent.key"
```

**Fallback zanjiri:** `locale` → `"uz"` → `message_key` o'zi.

### `localized_name(obj, locale)`

`name_uz` va `name_ru` ustunlari bo'lgan istalgan ORM modelida ishlaydi (T4 `Category`, `Product`, `PriceSegment` va boshqalar).

```python
from app.core.i18n import localized_name

# locale aniq ko'rsatilsa
localized_name(category, locale="ru")  # → category.name_ru

# locale=None — current_locale ContextVar'dan oladi
localized_name(product)  # → request tiliga mos nom

# name_ru bo'sh bo'lsa — name_uz ga fallback
localized_name(segment, locale="ru")  # → segment.name_uz (agar name_ru yo'q)
```

### `current_locale` ContextVar

```python
from app.core.i18n import current_locale

# Joriy tilni olish (middleware o'rnatadi)
locale = current_locale.get()  # → "uz" yoki "ru"

# Testda to'g'ridan-to'g'ri o'rnatish
token = current_locale.set("ru")
try:
    ...  # "ru" kontekstida ishlaydi
finally:
    current_locale.reset(token)
```

## 6. Yangi modul uchun i18n qo'shish (T4/T5 uchun qo'llanma)

T4 (Katalog), T5 (Mijoz bazasi) va keyingi modullar quyidagi ikki qadamni bajaradi.

### Qadam 1 — `messages.py` ga kalit qo'shish

`MESSAGES` dict ga yangi guruh qo'shing:

```python
# app/core/messages.py → MESSAGES dict ichida
"catalog.product_not_found": {
    "uz": "Mahsulot topilmadi",
    "ru": "Товар не найден",
},
"catalog.category_not_found": {
    "uz": "Kategoriya topilmadi",
    "ru": "Категория не найдена",
},
```

### Qadam 2 — `AppError` sub-sinf yoki to'g'ridan-to'g'ri ko'tarish

```python
from app.core.errors import AppError

# To'g'ridan-to'g'ri ko'tarish
raise AppError("catalog.product_not_found", status_code=404)

# Sub-sinf (ixtiyoriy, agar standart status kodi kerak bo'lsa)
class CatalogAppError(AppError):
    def __init__(self, message_key: str, status_code: int = 404, **kwargs):
        super().__init__(message_key, status_code=status_code, **kwargs)

raise CatalogAppError("catalog.product_not_found")
```

Global `AppError` handler (`main.py` da) avtomatik lokalizatsiya va envelope formatida qaytaradi. Qo'shimcha handler yozish shart emas.

### ORM modeli uchun `localized_name`

T4 da `Category` va `Product` modellari `name_uz`, `name_ru` ustunlariga ega bo'ladi. Javobda:

```python
from app.core.i18n import localized_name

# Schema/serializer ichida
{
    "name": localized_name(product),  # joriy request tiliga mos
    "name_uz": product.name_uz,       # ixtiyoriy — to'liq ma'lumot uchun
    "name_ru": product.name_ru,
}
```

## 7. Arxitektura qarorlari

**`ContextVar` + middleware** — FastAPI async kontekstda request izolyatsiyasi uchun. Thread-local o'rniga `ContextVar` ishlatiladi (asyncio Task darajasida izolyatsiya).

**`message_key` birinchi** — xatolar `{"detail": "Xato matni"}` o'rniga mashine-o'qiladigan `message_key` bilan qaytadi. Bu UI/mobil ilovalarni serverdan mustaqil lokalizatsiya qilishga imkon beradi.

**`Accept-Language` DoS himoyasi** — 256 belgidan uzun header regex ga uzatilmasdan rad etiladi.

**422 input echo yo'q** — Pydantic xato ob'ektidan `input`, `url`, `ctx` maydonlari serverda filtrlangan holda chiqariladi (parol/token qiymatlari response da ko'rinmaydi).

**500 trace yo'q** — `unhandled_exception_handler` faqat `common.internal_error` envelope qaytaradi; to'liq stack trace faqat server logida yoziladi.
